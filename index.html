แบบฝึกหัด Computer Network

1. Ethernet LAN

1.1 2 ระดับการสื่อสาร

  • - : ใช้สำหรับสื่อสารระหว่างการ์ดแลน NIC เครือข่ายเดียวกัน
  • - : ระบุเส้นทางส่งแพ็กเกจ

1.2 การส่งข้อมูล

  • ใช้ MAC address ของปลายทาง
  • เครื่องต้นทางส่งข้อมูลไป โดยใช้ MAC Address ของ Gateway แทน

1.3 การทำงานของ ARP

  • หรือ ใช้ใน IPv4 เพื่อหาว่า IP ของ Gateway (R1) มี MAC Address อะไร และใช้สร้าง/อัปเดตตารางจับคู่ที่เรียกว่า
  • อุปกรณ์จะทำการส่ง (แบบ )
  • อุปกรณ์ปลายทางที่มี IP ตรงกันจะตอบกลับมาด้วย (แบบ )
  • การบรอดแคสต์ ARP มากเกินไปอาจทำให้ประสิทธิภาพเครือข่ายลดลง และอาจเกิดการโจมตีแบบ

1.4 การทำงานบน IPv6

  • ทำหน้าที่แบบเดียวกับ ARP แต่ใช้สำหรับ IPv6
  • ใช้ข้อความ เรียกว่า เพื่อหา MAC Address เทียบได้กับ ARP ส่งแบบ
  • และ เรียกว่า เทียบได้กับ ARP ส่งกลับมา

2. IPv4 และ Subnet

2.1 โครงสร้างและการคำนวณ

  • IPv4 เป็นชุดตัวเลข บิต ที่แบ่งออกเป็น 2 ส่วน คือส่วน และส่วน
  • ใช้เป็นตัวกำหนดว่าส่วนไหนคือ และส่วนไหนคือ
  • ใช้/ไว้ เพื่อบอกว่า network มีกี่ bit
  • Prefix ตัวเลขยิ่งมาก (เช่น /30) - Network ยิ่ง
  • Prefix ตัวเลขยิ่งน้อย (เช่น /8) - Network ยิ่ง
  • คือการนำ IP Address มาคำนวณทางตรรกะศาสตร์ (AND) กับ เพื่อหา Network Address
  • ประเภทของ Address ใน 1 เครือข่ายจะประกอบด้วย , , และ

2.3 รูปแบบการส่งข้อมูล

  • : การส่งข้อมูลจาก 1 เครื่องไปยังอีก 1 เครื่อง (1 to 1)
  • : การส่งข้อมูลจาก 1 เครื่องไปยังทุกเครื่องในเครือข่าย (1 to All)
  • : การส่งข้อมูลจาก 1 เครื่องไปยังกลุ่มเป้าหมายเฉพาะกลุ่ม (1 to Many)

2.6 อุปกรณ์และการแบ่งเครือข่าย (Subnet)

  • : เมื่อได้รับ Broadcast มันจะ "ส่งต่อออกทุกพอร์ต" (ยกเว้นพอร์ตที่ส่งมา)
  • : จะ "ไม่ส่งต่อ" (Stop/Block) ข้อมูลที่เป็น Broadcast ข้ามไปยังฝั่งอื่น
  • เหตุผลที่ต้องทำ Subnet: เพื่อลดขนาดของ
  • Router คืออุปกรณ์ที่ช่วยแบ่ง และเป็นตัวหยุดไม่ให้ Broadcast ทะลุข้าม
  • การทำ Subnet คือการยืมบิตจากฝั่ง Host มาทำเป็น Network ทำให้จำนวนเครือข่าย ย่อยเพิ่มขึ้น แต่จำนวน IP ที่แจกให้ ในแต่ละเครือข่ายจะลดลง

3. IPv6

3.1-3.2 การทำงานของ IPv6

  • เทคนิคในการเปลี่ยนผ่านเครือข่าย มี 3 รูปแบบหลัก ได้แก่
  • 1. (เปิดให้รันทั้ง IPv4 และ IPv6 พร้อมกัน)
  • 2. (ห่อหุ้มแพ็กเก็ต IPv6 ไว้ในแพ็กเก็ต IPv4)
  • 3. (ใช้ NAT64 ในการแปลที่อยู่)
  • IPv6 มีความยาว บิต
  • แบ่งรูปแบบการส่งเป็น 3 ประเภทหลักคือ , และ
  • หรือ : ทำหน้าที่เหมือน Public IP ใน IPv4
  • หรือ : จำเป็นต้องมีสำหรับทุกอุปกรณ์ในเครือข่ายย่อยเดียวกัน

4. Transport Layer

  • หรือ เน้นความน่าเชื่อถือผ่านกระบวนการ
  • หรือ เน้นความรวดเร็วและ Overhead ต่ำ
  • เมื่อนำ IP Address มารวมกับหมายเลขพอร์ต จะเรียกว่า
  • (0-1,023) Port : สงวนไว้สำหรับบริการทั่วไป เช่น HTTP
  • (1,024-49,151) Ports : พอร์ตที่ลงทะเบียนไว้เฉพาะทาง
  • (49,152-65,535) Ports : สุ่มเลือกให้ไคลเอนต์ใช้ชั่วคราว

5. Application Layer

  • Layer: อินเทอร์เฟซระหว่างแอปพลิเคชันกับระบบเครือข่าย
  • Layer: จัดรูปแบบข้อมูล บีบอัด และเข้ารหัส/ถอดรหัส
  • Layer: สร้าง รักษา และเริ่มเซสชันการสื่อสาร
  • Model: อุปกรณ์ขอบริการคือ Client ให้บริการคือ Server
  • Model: ทำหน้าที่เป็นทั้ง Client และ Server ในเวลาเดียวกัน
  • หรือ : แปลงชื่อโดเมนเนมให้เป็น IP Address
  • หรือ : แจกจ่าย IP, Subnet mask อัตโนมัติ
  • หรือ : ดึงหรืออัปโหลดข้อมูล
  • หรือ : โปรโตคอลแชร์ไฟล์แบบไคลเอนต์/เซิร์ฟเวอร์

6. Security

  • : ซอฟต์แวร์ประสงค์ร้าย แบ่งเป็น
  • ต้องอาศัยไฟล์โฮสต์หรือคนช่วยรัน
  • แพร่กระจายตัวเองได้โดยอิสระ
  • หลอกว่าเป็นโปรแกรมปกติและต้องพึ่งพาให้ผู้ใช้หลงกล
  • การโจมตีเครือข่าย 3 หมวดหมู่หลัก
  • : การสอดแนมเพื่อหาข้อมูล (เช่น ping, whois)
  • : โจมตีเพื่อเข้าถึงข้อมูล (เจาะรหัส, Man-in-the-middle)
  • : โจมตีทำให้ระบบล่ม (Botnet/Zombies รุมโจมตีพร้อมกันคือ DDoS)
